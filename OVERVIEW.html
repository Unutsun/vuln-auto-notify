脆弱性自動通知システム

IPA（MyJVN）の脆弱性情報と、社内アプリ一覧（資産管理ツールのCSV）を突き合わせて、該当があれば Teams に自動通知する仕組み。

● 動作確認ずみ（通知の実送信まで成功）

1. 全体の構造（4つの部品が順番に動く）

各部品は独立したプログラム。あいだは「ファイル（JSON）」だけでつながる＝疎結合。1個ずつ直せる・テストできる。

1fetch_vulnsIPAから脆弱性を取得→ vulns.json
→
2load_assets社内アプリCSVを読む→ assets.json
→
3match突合（本体）→ alerts.json
→
4notifyTeamsへ送信→ Teams

ぜんぶまとめて動かす司令塔 = run_all。Windowsのタスクスケジューラで毎日1回呼ぶ想定。

2. それぞれの部品の役割

部品やること入口 → 出口
fetch_vulnsIPAの公式API(MyJVN)から新しい脆弱性を取る。CVSS7.0未満は捨てるMyJVN API → vulns.json
load_assets資産管理ツールの社内アプリCSVを読む。列が増減してもOK（ヘッダ名で対応）CSV → assets.json
match脆弱性×社内アプリを突合。名寄せ表(別CSV)経由で「社内アプリ＝log4j」等を判定。確度(高/中/低)と根拠つき2つのjson → alerts.json
notify当たったものをTeamsへ送信。同じものは二度送らない。送信先URLは設定で差し替えalerts.json → Teams
正本ドキュメント = README.yaml 設定 = config.yaml 送信先URL = .env テスト = tests/（全37件パス）

3. これからやること

  • 仕組みを4部品で実装（fetch / load / match / notify / run_all）
  • テスト全37件パス・実データ(IPA 121件×社内15件→104件ヒット)で動作確認
  • 通知の実送信テスト成功（webhook.site に3件届いた）
  • 【次やる】会社（職場）のTeamsで「ワークフロー」を作り、送信先URLをもらう
    ※個人/コミュニティTeamsには機能が無い。会社のMicrosoft365アカウントが必要。
  • 【次やる】.env のURLを、テスト用から会社TeamsのURLに書き換える（コードは触らない）
  • 本物の資産管理ツールのCSVを input/ に置き、config.yaml の列名を実物に合わせる
  • 名寄せ表(product_aliases)を実環境に合わせて少しずつ育てる
  • Windowsタスクスケジューラで毎日自動実行に登録

判断ペンディング（今は「全件通知」のまま）： ダミーだとChromeだけで約100件ヒットする。本番で多すぎたら「自動更新される製品(Chrome等)を対象外にする」「製品ごと1通にまとめる」「CVSS9以上だけ」等で絞れる。必要になったら設定で対応。
